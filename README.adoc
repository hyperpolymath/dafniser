== What Is This?

Dafniser generates **correct-by-construction code** for critical functions
using Microsoft's Dafny verification language.

Dafny can prove that code satisfies its specification, but nobody outside
Microsoft Research uses it. Dafniser makes verified code generation accessible.

== How It Works

Annotate critical functions with pre/postconditions in `dafniser.toml`. Dafniser:

1. Generates Dafny code with your specifications as formal contracts
2. Verifies the code satisfies all contracts (compile-time proof)
3. Compiles verified Dafny to C#, Go, Java, or JavaScript
4. Creates a Zig FFI bridge to integrate with your existing code

== Key Value

* **Proven-correct code** for security-critical functions
* **Auto-generated from specifications** — no manual Dafny
* **Multiple output languages** — verified code in your ecosystem

== Architecture

Follows the hyperpolymath -iser pattern (same as https://github.com/hyperpolymath/chapeliser[Chapeliser]):

* **Manifest** (`dafniser.toml`) — describe WHAT you want
* **Idris2 ABI** (`src/abi/`) — formal proofs of interface correctness
* **Zig FFI** (`ffi/zig/`) — C-ABI bridge to target language runtime
* **Codegen** (`src/codegen/`) — generates target language wrapper code
* **Rust CLI** — parses manifest, validates, generates, builds, runs

User writes zero target language code. Dafniser generates everything.

Part of the https://github.com/hyperpolymath/iseriser[-iser family] of acceleration frameworks.
